BANK MARKETING DATASET ANALYSIS¶

Author: Ioannis Frangistas (Copyrighted Material - All copyrights reserved)

Overview & Scope¶

Business Problem¶

A bank conducts telephone sales campaigns aimed at convincing customers to open a term deposit. This analysis seeks to:

  • Accurately predict which customers are likely to respond positively.
  • Understand the most important factors influencing that decision.
  • Optimize the targeting of future campaigns by reducing call costs.

Analytical Objectives¶

  1. Data Cleaning & Preprocessing — Clean raw data, handle missing values, apply outlier capping and one-hot encoding.
  2. EDA — Analyze distributions, class imbalance, and relationships between variables.
  3. Model Training & Comparison — Train and compare Logistic Regression, Random Forest, and XGBoost.
  4. Model Selection — Select the best model based on ROC AUC, Average Precision, and F1.
  5. Explainability — Interpret results using Feature Importances and SHAP values.

Dataset Description¶

Attribute Value
Source UCI Bank Marketing — bank-additional-full.csv
Original size 41,188 records × 21 columns
After cleaning 41,170 records × 20 columns
Target deposit (1 = yes, 0 = no)
Class distribution ~89% No (36,548) / ~11% Yes (4,640)

Feature Groups¶

  • Demographic: age, job, marital, education
  • Financial: housing, loan
  • Campaign: contact, month, day_of_week, duration, campaign, previous, poutcome, prev_contact (engineered feature)
  • Macroeconomic indicators: emp.var.rate, cons.price.idx, cons.conf.idx, euribor3m, nr.employed

Executive Summary¶

Key EDA Findings¶

  • Strong class imbalance: Only ~11% of customers subscribe to a term deposit, requiring class_weight='balanced' and evaluation metrics beyond simple accuracy.
  • Call duration (duration): An extremely powerful predictor — longer calls are strongly associated with positive responses, though this variable is only known after the call takes place.
  • Macroeconomic context: euribor3m and cons.price.idx significantly influence subscription probability — periods of lower interest rates tend to favor deposits.
  • Timing: May calls show a negative effect, while October and March are associated with higher conversion rates.
  • Previous contact outcome (poutcome_success): Customers with a successful response in a prior campaign are far more likely to subscribe again.

Conclusion¶

XGBoost achieves ROC AUC ≈ 0.95 — excellent performance for highly imbalanced marketing data. SHAP analysis shows that call duration and macroeconomic conditions dominate the predictions. Campaign timing (avoiding May, focusing on October/March) and leveraging customer history (poutcome_success) can significantly improve the effectiveness of the next campaign.

# Helper: count class instances
def class_counts(y_vec):
    unique, cnt = np.unique(y_vec, return_counts=True)
    d = {int(k): int(v) for k, v in zip(unique, cnt)}
    return d.get(0, 0), d.get(1, 0)

# Random Forest classifier
rf_clf = RandomForestClassifier(
    n_estimators=300,
    max_depth=None,
    min_samples_leaf=50,
    class_weight="balanced",
    random_state=42,
    n_jobs=-1,
)

# XGBoost classifier with scale_pos_weight for imbalance
pos_weight = (len(y_train) - y_train.sum()) / y_train.sum()
xgb_clf = XGBClassifier(
    n_estimators=400,
    learning_rate=0.05,
    max_depth=4,
    subsample=0.8,
    colsample_bytree=0.8,
    objective="binary:logistic",
    scale_pos_weight=pos_weight,
    eval_metric="logloss",
    random_state=42,
    n_jobs=-1,
)

def evaluate_model(name, model, X_train_in, X_test_in, y_train_in, y_test_in):

# Model Fit
    model.fit(X_train_in, y_train_in)
    y_proba = model.predict_proba(X_test_in)[:, 1]

    roc_auc = roc_auc_score(y_test_in, y_proba)
    ap = average_precision_score(y_test_in, y_proba)

    thresholds = np.linspace(0.1, 0.9, 81)
    f1_scores = []
    for t in thresholds:
        y_pred = (y_proba >= t).astype(int)
        f1_scores.append(f1_score(y_test_in, y_pred))

    best_idx = int(np.argmax(f1_scores))
    best_t = thresholds[best_idx]
    best_f1 = f1_scores[best_idx]

    return {
        "name": name,
        "model": model,
        "roc_auc": roc_auc,
        "avg_precision": ap,
        "best_threshold": best_t,
        "best_f1": best_f1,
        "y_proba": y_proba,
    }
